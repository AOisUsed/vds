
## (极其简短的)介绍
此系统的功能是进行设备间通信，vds表示一个设备集群，内部有多个设备，以及一些与通信、路由等相关的模块。


## 开始使用
### 一次简单的通信 
以下流程具体代码操作见 `vds_test.go` 文件中 `TestBasicCommunication()` 测试,
流程图见`vds图示` 中`消息通信流程`tab

准备：
1. 创建公用数据源作为注册中心以供多个vds查询设备物理通讯信息
2. 建立一个或多个vds
3. 在vds中创建一个或多个vd,并把vd的连接方式注册到注册中心中，启动vd服务


接收方：
4. 在3步中调用`ConnectAndRegisterDevice()`后设备已经进入接收状态，可以接收消息了

发送方：
5. 提供目标设备的id,消息传输内容,调用vd的`Send()`方法,发送消息

结束：
6. 完成流程后调用`vds.Stop()`会停止所有vds中的设备和正在运行的模块并退出。

## VDS 模块
### Ingress Router
#### 生命周期
- 启动：
  - `Run()`:开启监听上游消息 
- 终止：
  - 上游消息通道关闭，自然终止
  - `Stop()`：1. 结束监听上游消息(强制性，会使上游发送通道阻塞) 2. 关闭所有路由出口消息通道
  

----

### Virtual Device
#### 生命周期
- 启动：
  - `Run()`： 开启监听上游消息
- 终止:
  - `Stop()` ：1. 结束监听上游消息 (强制性，会使上游发送通道阻塞)。2.关闭下游消息通道


----

###  Aggregator
#### 生命周期
- 启动：
  - `Watch()`：开启监听某一个上游消息通道，适用于服务运行中动态添加 
  - `Run()`：开启监听一群上游消息通道，适用于服务初始化时添加(也可以在运行时调用)
- 终止：
  - `Stop()` ：在等待上游所有消息通道结束后，再关闭下游消息通道(非强制性终止)


----

### Dispatcher
#### 生命周期
- 启动：
  - `Run()`：启动工作池，开始监听上游消息
- 终止：
  - 上游消息通道关闭，自然终止
  - `Stop()`：结束监听上游消息(强制停止，会使上游发送通道阻塞)
